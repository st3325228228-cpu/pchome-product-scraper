# PChome Product Scraper

使用 Python、Streamlit、pandas 與 Plotly 建立的 PChome 商品資料蒐集與價格分析儀表板。

使用者可以透過商品關鍵字或 PChome 特設商品分類取得商品資料，系統會分析商品價格、品牌、價格區間、原價、特價與折扣幅度，並提供互動式圖表、自選商品比較及 CSV 資料匯出功能。

本專案主要用於展示 Python 網路資料存取、JSON API 串接、資料清洗、價格分析、資料視覺化、Session State 狀態管理及 Streamlit Web App 部署能力。

## 主要功能

### 1. 商品搜尋模式

系統提供兩種主要資料取得模式：

```text
關鍵字搜尋
PChome 特設分類
```

#### 關鍵字搜尋

使用者可輸入商品關鍵字，例如：

```text
耳機
```

```text
筆記型電腦
```

```text
無線滑鼠
```

```text
機械式鍵盤
```

```text
智慧手錶
```

系統會透過 PChome 商品搜尋資料介面取得符合條件的商品。

#### 特設分類

系統提供多種 PChome 特設商品分類，例如：

- 耳機音響特賣
- 筆記型電腦特賣
- 桌上型電腦與螢幕特賣
- 手機特賣
- 鍵盤與滑鼠週邊特賣
- 印表機特賣
- 相機特賣
- 電競週邊特賣
- 行動電源特賣
- 穿戴裝置特賣
- 喇叭特賣
- 智慧家電特賣
- 家電特賣
- 冷氣特賣

特設分類可能額外提供：

- 商品原價
- 商品特價
- 折扣幅度
- 節省金額

系統提供常用關鍵字快速選單：

- 耳機
- 筆電
- 滑鼠
- 鍵盤
- 手機
- 相機
- 螢幕

- 智慧燈泡

使用者也可以選擇：

搜尋其他商品。

---

### 3. 商品抓取數量

系統依照使用者設定的目標筆數，自動計算需要取得的頁數。

每一頁大約取得：

```text
20 筆商品
```

程式會持續取得資料，直到：

- 達到使用者指定的商品數量
- 搜尋結果沒有更多商品
- 外部資料請求失敗

---

### 4. 資料擷取進度

資料取得期間會顯示：

- 目前頁數
- 總頁數
- 已取得商品數量
- 整體處理進度

例如：

```text
第 2／5 頁 · 40 筆
```

完成後，進度列會自動移除。

---

### 5. 商品基本資料

關鍵字搜尋模式主要取得：

- 商品名稱
- 商品售價
- 商品 ID
- 商品品牌
- 是否為特設商品

特設分類模式可能另外取得：

- 商品原價
- 商品特價
- 折扣百分比

商品資料會整理成 pandas DataFrame，供後續統計與圖表使用。

---

### 6. 商品價格統計

系統會自動計算：

- 商品總數
- 平均價格
- 最高價格
- 最低價格
- 價格中位數
- 價格標準差

這些統計結果可以快速呈現商品市場的價格分布情況。

---

### 7. 商品價格區間

商品會依照售價分類至不同價格區間：

| 價格範圍 | 價格區間 |
|---:|---|
| 500 元以下 | 超值區 |
| 500～999 元 | 經濟區 |
| 1,000～1,999 元 | 中價區 |
| 2,000～4,999 元 | 高價區 |
| 5,000 元以上 | 頂級區 |

價格區間可用於：

- 商品分類
- 圖表顯示
- 商品篩選
- 自選商品比較

---

### 8. 品牌資料

系統會取得 PChome 商品資料中的品牌欄位。

可以用於分析：

- 各品牌商品數量
- 各品牌平均價格
- 各品牌價格範圍
- 品牌與價格區間關係

部分商品可能沒有完整品牌資料，實際結果以 PChome 回傳內容為準。

## 特設商品分析

當特設分類提供原價與特價資料時，系統會開啟進階折扣分析。

### 1. 折扣摘要

系統會顯示：

- 有折扣商品數
- 平均折扣幅度
- 最高折扣幅度
- 全部商品總節省金額

節省金額計算方式：

```text
原價－特價
```

---

### 2. 折扣排行榜

系統會選出折扣幅度最高的商品，建立 Top 15 排行榜。

排行榜顯示：

- 商品名稱
- 折扣百分比
- 可節省金額
- 折扣強度顏色

商品依照折扣幅度由高到低排列。

---

### 3. 原價與特價比較

系統使用散佈圖比較：

```text
X 軸：商品原價
Y 軸：商品特價
```

圖表加入無折扣基準線。

若商品位置越低於基準線，代表原價與特價的差距越大。

---

### 4. 折扣幅度分布

系統使用直方圖顯示商品折扣幅度分布，可觀察：

- 商品主要折扣範圍
- 高折扣商品數量
- 低折扣商品數量
- 折扣幅度是否集中

---

### 5. 特設商品明細

特設商品明細包含：

- 商品名稱
- 特價
- 原價
- 折扣百分比
- 品牌

資料會依照折扣幅度由高到低排序。

---

### 6. 特設商品 CSV

使用者可將特設商品資料下載為 CSV。

檔名格式例如：

```text
PChome_特設_20260705_1530.csv
```

CSV 使用：

```text
UTF-8-SIG
```

可降低使用 Microsoft Excel 開啟中文內容時出現亂碼的機率。

## 自選商品比較

### 1. 加入自選商品

使用者可以從商品搜尋結果選擇需要比較的商品。

選取結果會儲存在：

```text
Streamlit Session State
```

在目前操作階段切換頁面或圖表時，自選商品不會立即消失。

---

### 2. 自選商品摘要

比較面板會顯示：

- 已選商品件數
- 最低價格
- 最高價格
- 平均價格

---

### 3. 自選商品卡片

每個自選商品會顯示：

- 商品名稱
- 商品價格
- 價格區間
- 是否為特設商品
- 折扣幅度
- 商品價格高於或低於平均價多少

---

### 4. 橫向長條圖

系統使用橫向長條圖比較自選商品價格。

若商品具有原價資料，圖表會同時顯示：

- 商品原價
- 商品售價

圖表也會標示全部搜尋結果的平均價格。

---

### 5. 散佈比較圖

散佈圖會同時顯示：

- 全部搜尋商品
- 自選商品

自選商品會使用較明顯的圖示標記，方便觀察其在整體價格分布中的位置。

---

### 6. 自選商品資料表

使用者可查看自選商品完整資料，例如：

- 商品名稱

## 快取設定

關鍵字搜尋與特設分類資料使用：

```python
@st.cache_data(ttl=600)
```

代表相同條件的資料會快取約：

```text
600 秒
10 分鐘
```

快取可以：

- 降低相同搜尋的重複請求
- 改善載入速度
- 減少 PChome 資料介面的負擔
- 改善使用者操作體驗

---

## 請求控制

商品搜尋請求的逾時設定為：

```text
10 秒
```

不同搜尋頁面之間會暫停約：

```text
0.8 秒
```

這些設定可以：

- 降低連續請求頻率
- 避免單次請求長時間等待
- 降低外部服務壓力
- 改善錯誤處理

---

## 錯誤處理

當某一頁資料取得失敗時，系統會顯示：

```text
第 N 頁失敗
```

或：

```text
特設第 N 頁失敗
```

其他已成功取得的商品資料仍可繼續分析。

常見錯誤來源包括：

- 網路連線失敗
- 請求逾時
- PChome API 格式調整
- JSON 欄位不存在
- 特設分類沒有商品
- 商品資料不完整

## 系統限制

- PChome 商品資料介面並非由本專案控制。
- PChome 修改 API、欄位或網站結構後，程式可能需要更新。
- 搜尋結果不一定包含 PChome 網站上的所有商品。
- 商品售價可能因促銷活動、會員資格或優惠券而不同。
- 商品原價與特價資料不一定每筆都有。
- 部分特設分類可能無法取得折扣資訊。
- 部分商品可能沒有品牌資料。
- 商品資料可能包含重複或相似商品。
- 商品庫存、圖片、評價與配送資訊可能未完整取得。
- 自選商品只保留在目前 Streamlit Session。
- 重新整理頁面後，自選商品可能消失。
- 免費 Streamlit 部署可能進入休眠。
- 大量或高頻率查詢可能受到外部服務限制。

---

## 資料使用注意事項

- 請避免短時間內大量或高頻率取得資料。
- 請遵守 PChome 網站及相關服務的使用規範。
- 商品名稱、價格、折扣與庫存應以 PChome 官方頁面為準。
- 本專案不保證資料即時性、完整性或永久可用性。
- 不應將本專案用於未經授權的大規模資料蒐集。
- 公開部署時應設定合理的快取及查詢限制。
